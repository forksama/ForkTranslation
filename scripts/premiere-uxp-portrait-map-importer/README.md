# Premiere UXP Portrait Map Importer

Development UXP panel for placing standing portrait images onto a Premiere Pro
video track from a ForkTranslation `voice-portrait-map.json`.

The mapping file is generated by the GPT-SoVITS and VOICEVOX voice workflows.
The panel matches existing audio clips in the active Premiere sequence by audio
file basename, imports the mapped portrait images, places them on a target video
track, extends each portrait through the gap before the next audio clip, and
applies a bottom-aligned Motion transform. The final portrait ends with its own
audio clip.

## Load

1. Open Premiere Pro 25.6 or newer.
2. Enable Developer Mode in Premiere Pro preferences and restart Premiere.
3. Open Adobe UXP Developer Tool.
4. Add this plugin manifest:
   `scripts/premiere-uxp-portrait-map-importer/manifest.json`
5. Load the plugin, then open the panel named `Portrait Map Importer`.

## Use

1. Activate the target Premiere sequence.
2. Choose `media/voice/voice-portrait-map.json`.
3. Confirm or choose the thread workdir. The panel can infer this from
   `<workdir>/media/voice/voice-portrait-map.json`.
4. Choose Portrait Dir. This must use the same logical base level as GPT-SoVITS
   and VOICEVOX used when writing `portraitRelPath`.
5. Enter the source audio track in `A1` format.
6. Choose the target video track mode. `new` creates the next video track;
   `auto` uses the first existing video track with no clip items or creates a
   new one if none are empty.
7. Click `Scan` to validate, then `Import Portraits`.

Existing target video tracks must be empty. The panel does not remove or replace
existing clips.

## Notes

- Matching is by exact `audioFileName` basename.
- Mapping files whose item fields use `audioRelPath` and `portraitRelPath` are
  supported even if their `schemaVersion` label is older.
- Portrait images are scaled proportionally so their full source image height is
  95% of the sequence frame height, then anchored to the bottom center.
- Each portrait starts with its mapped audio and ends at the next mapped audio's
  start. The final portrait ends with its own mapped audio.
- The panel verifies placed portrait timing after trimming and logs per-clip
  target vs portrait start/end/duration differences.
- If Premiere accepts placement but a trim or Motion action does not verify, the
  panel logs a warning.
