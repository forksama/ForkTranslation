# 学马仕翻译风格指南

- 建档日期：2026-07-13
- 状态：首版
- 上游规格：[Research-004 §2](../../../docs/research-004-gakumas-thread-translation-formats-and-knowledge-layout.md#2-翻译产品优先级) 与 [Research-005 §6](../../../docs/research-005-gakumas-stable-knowledge-first-version-spec.md#6-style-guidemd-首版规格)

本文档定义 `domains/gakumasu/` 领域内所有翻译（B 完整译文、C 字幕 cue）都必须遵守的风格约束。规则冲突时按 §1 优先级判定。

## 1. 忠实度优先级

按以下顺序理解"忠实"：

1. **忠实于人物口吻**——优先保留角色、发帖者或叙述者的语气、情绪强度、说话节奏、距离感和故意的不自然。
2. **忠实于中文互联网阅读体验**——在不改变含义的前提下，让中文读者读起来像自然的中文社区内容或自然的视频字幕。
3. **忠实于字面**——保留原文信息、指代和逻辑，但不机械照搬日语语序。

三条硬约束：

- **不加戏**：不新增原文没有的事实、心理活动、笑点、铺垫或解释。
- **不擅自删减**：被选入 A 的楼层必须在 B 中有对应译文；确实不适合上屏的内容必须在 `备注` 中说明原因。
- **不替原帖作者纠错**：原帖误读剧情或表达含混时，译文应保留这种状态。

## 2. 语感基线

- 译文中文语感**向日语宅圈 / 汉化组风格靠**。允许保留少量日式表达和宅圈术语不译，允许必要时加译注。
- **不追求脱日化的自然中文**。翻译对象是学马仕日文社区讨论，脱日化会丢掉原本的圈层氛围。
- **不采用 B 站弹幕 / 微博风格**。避免"绝绝子""芭比 Q 了""破防了"等强中文互联网时代梗——它们会替原文加戏。
- 简体中文优先。术语与人名与 [`glossary.csv`](glossary.csv) 一致。

## 3. 称呼与敬语

### 3.1 P 的称呼

- `プロデューサー / P → 学P`（作为角色称呼固定）。所有 C 字幕的 role 字段中制作人角色统一写作 `学P`，不要出现 `P` 或 `制作人` 作为 role 名。
- 对话正文中角色以名字或亲近方式称呼 P 时，可用 `制作人` 或原文昵称对应译法。
- P 自称、旁白提及、以及其他角色以泛称提及 P 时，一律用 `学P`。

### 3.2 日文后缀

日文人名后缀 `さん / 様 / ちゃん / 先輩 / 君` 等**全部本土化**为中文习惯，不保留原日文后缀：

- `〇〇さん` → `〇〇小姐` / `〇〇同学` / 直呼名字（视上下文和亲疏度）
- `〇〇様` → `〇〇大人` / `〇〇小姐` / 敬体表达
- `〇〇ちゃん` → `小〇〇` / `〇〇酱`（后者仅在角色本人在原文中用了明显的宅圈风撒娇口吻时才使用）
- `〇〇先輩` → `〇〇前辈`
- `〇〇君` → `〇〇` / `小〇〇`（视上下文）

上述映射不是硬性表；具体取舍以"角色关系与情绪距离"为准，允许在 `translation-decisions.md` 中登记角色对特定人的固定称呼。

### 3.3 官职

学园长、秘书、班主任、训练员等官职**一律中文化**。学园长不译作"学园长（sensei）"或保留日文。

### 3.4 昵称

角色间稳定的昵称（如日文的爱称）：

- 有中文社区共识译法的（进入 `glossary.csv` 或 `community-expressions.md`）按共识译法。
- 没有共识的按 §3.2 规则处理；若在同一次翻译内需要固定，登记到该串的 `context-pack.md`。

## 4. 粗口与强语气

**策略：偏向软化但不过度**，按上下文情景判断强度。

- `バカ / ばか`：视语气强度译作**「笨蛋」「蠢货」等中等强度**中文词。不译作"傻瓜"这类过软表达，也不拔高到"废物"以上。
- 更强等级（`クソ / 死ね / うぜぇ` 等）：视具体情景判断，允许保留粗口力度，**不硬性统一**。
- 角色本身情绪爆发场景（比如燐羽被戳到痛处的失态）优先保留强度，不因"字幕美观"软化。
- 反讽、玩梗式使用的粗口（比如朋友间的"バカ"）反而应软化，以匹配语境。

## 5. 语气词

日文句尾 `ね / よ / な / わ / かな / だろ / でしょ` 等**不逐一映射**到中文。

- 目标："中文读起来自然"，句尾语气**靠句式、标点、词序**传达，只在必要时加"呢""吧""啊""吗"。
- 忌讳：`〜だよね` 一律译作"是吧呢"这类叠加语气词的机械翻译。
- 每个角色的固定句尾风格（如燐羽偏冷淡直白、清夏偏温柔含糊）在对应角色 profile 或 `characters.md` 中登记，翻译时按角色调优句式而不是套模板。

## 6. 符号规则

### 6.1 对话引号

**保留 `「」`**（与 A/B/C 中 `角色名「台词」` 的写法统一，便于机器拆分）。**不**替换为中文双引号 `""`。

### 6.2 省略号

统一使用中文全宽 `……`（U+2026 × 2）。

- 不使用日文单字符 `…`（U+2026 × 1）。
- 不拓展为 `…………` 等长省略号；如需表达特别长的沉默，改用文字描述或 `……(长时间沉默)……` 类叙述性表达。

### 6.3 中黑符号

- 人名中的中黑 `・` **不保留**，改用中点 `·`（如 `葛城·Lilja` 而非 `葛城・リーリヤ`）；简单情况可直接用空格 `葛城 Lilja`。
- 组合名内部的 `・` 视排版需要保留或改 `·`，不硬性统一。

### 6.4 书名号 / 作品名号

**按原文保留，不做统一化替换。**

| 原文符号 | 译文符号 | 场景 |
|---|---|---|
| `『』` | `『』` | 歌曲名、专辑名、作品名、剧情章节名等 |
| `「」` | `「」` | 对话、强调、引用等 |
| `《》` | `《》` | 原文使用中文书名号（罕见） |
| `""` `""` `""` | 各自沿用 | 原文使用引号 |

**不**引入中文书名号 `《》` 去覆盖原文的 `『』`，也**不**把 `『』` 统一改成 `《》`。

## 7. 梗与反讽

- **优先保留日文原梗**。字幕或译文中必要时加译注。
- **不主动替换**成中文圈现有梗。例如 `曇らせ` 不译作"破防"。
- **不为了流畅擅自脑补解释**。原文只写了梗的表层，译文也只保留表层；深层含义作为审校备注或译注，不进正文。
- 无法判断是否为梗时按"不加戏"原则**直译**，并在 `translation-B.md` 的 `备注` 中标注为疑点。
- 已建立中文对应的社区表达进入 `community-expressions.md`。

## 8. 译注

- 译注**不进入 B 的正文**，作为 `备注` 字段附加。
- C（字幕）**默认不放译注**。如剧情理解绝对依赖某个日文梗，字幕上可以出现极短的括号说明（如 `（关西方言）`），但不能超过 6 个中文字符。
- 长解释一律不上屏；改为审校备注，或引导观众参考视频简介 / 弹幕。

## 9. B 与 C 的差异

- **B（`translation-B.md`）**：审校版。允许一句话译得较长、较完整，允许分段和插入`备注`。
- **C（`pr-subtitles-C.jsonl`）**：字幕 cue。必须按 [Research-004 §5.4](../../../docs/research-004-gakumas-thread-translation-formats-and-knowledge-layout.md#54-c-的分行与拆分规则) 切分，默认每 cue 1 行、最多 2 行；每行 15–20 个中文字符。
- **同一处翻译，B 和 C 可以措辞不同**：B 追求准确完整，C 追求节拍和可读性。C 允许在保留原意的前提下调整语序、去掉冗余助词。
- 从 B 到 C 的角色拆分：楼主正文中的叙述句拆为 `旁白`，角色台词拆为具体角色；同一 cue 内不混入多个角色。

### 9.1 B/C 对照实例

**首版留白。** 在第一个真实翻译项目完成后，回填一段完整实例：一段日文原文 + 对应的 B 版译文 + 对应的 C 版字幕 cue。这段实例作为"审校版 vs 字幕版"差异的锚点样例。

## 10. 与其他文件的分工

| 内容类型 | 归属 |
|---|---|
| 全项目通用译名 | [`glossary.csv`](glossary.csv) |
| 已作出的具体翻译决策（含冲突裁决） | [`translation-decisions.md`](translation-decisions.md) |

| 社区表达、黑话、梗 | [`community-expressions.md`](community-expressions.md) |
| 单串专用称呼与临时决策 | 对应 `threads/*/context-pack.md` |
