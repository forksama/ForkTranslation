本文档登记学马仕**系统级别的官方背景骨架**——学园结构、培育系统、亲爱度机制、组合模型、P × 偶像的契约模型——以及**主线剧情反复被提及的关键事件名**。目标是让翻译时遇到相关术语或事件引用时，能直接查到"这是什么、翻译时要如何判断"。

## 使用规则

- **只写官方设定**。二创共识不入本文件——二创层面的角色形象偏移放到 `character-profiles/*.md`，社区表达放到 [`community-expressions.md`](community-expressions.md)。
- **只写"影响翻译判断"的内容**。不复述 wiki，不逐字介绍系统机制。判断标准：这条信息若缺失，译者是否会在译文中做出错误的语气 / 用词 / 指代选择。
- **不复述角色个体信息**。人物地图见 [`characters.md`](characters.md)；本文件出现具体角色时只作**指代锚点**，不重复其性格 / 台词 / 关系。
- **现实运营层（直播、卡池、活动、周年周边）首版不写**。遇到具体串需要时，在该串的 `context-pack.md` 里临时补充。
- 系统术语的中文译名以 [`glossary.csv`](glossary.csv) 为准，本文件不重复登记译名。
- 学 P × 各偶像的关系（并行世界线）见 [`relationships.md §9`](relationships.md#九制作人学-p-关系)。

# 一、学园结构

## 1.1 初星学园 · 極月学园的对立

学马仕主舞台是两个偶像培育学园的对立结构：

| 学园 | 位置 | 定位 | 校长 |
|---|---|---|---|
| **初星学园**（はつぼし） | 主角方 | 日本最大的偶像培育学校；初等部 / 中等部 / 高等部 / 职业学院一体的私立学园 | 学园长：**十王邦夫** |
| **極月学园**（ごくげつ） | 对立方 | 顶级偶像培育强豪校；「極月」= 12 月的异称 / 冬之季语，与「初星」（新年季语）成对命名 | 校长：**黒井崇男**（原「偶像大师」系列 961 プロ 社长） |

**翻译判断**：

- `初星` / `極月` **不译**，作为专名保留（见 [`glossary.csv`](glossary.csv)）。
- `初星` 作为前缀词广泛出现：`初星学園` `初星水` `初星IDOL FESTIVAL` `初星乳清蛋白` `初星のギャル`（社区戏称）等，一律保留原字。
- `学園長` 译作**"学园长"**，不译"校长"（校长是纯行政词，学园长带更强的"教育机构总负责人"意味）。極月的黒井崇男译作**"校长"**因为极月称呼与初星不同（黑井有系列惯用形象），若原文明确写「学園長」则也用"学园长"。
- 「対立学園」译作**"敌对学园"**或"对立学园"，不译"敌对学校"。

## 1.2 学部与年级

初星学园高等部主要构成：

- **アイドル科**（偶像科）：主要培育对象所在的科，1 / 2 / 3 年三届。
- **普通科**：非偶像培育路径，人物如 `真城優`（2 年生放送部部长）在此。

1 / 2 / 3 年生的敬语强度差异见 [`characters.md`](characters.md#说话风格通用注意事项)（1 年生 15 岁 / 3 年生 17 岁）。

**翻译判断**：

- `アイドル科` 译**"偶像科"**（不译"艺人科""偶像专业"）；`普通科` 译**"普通科"**（不译"普通班""普通专业"）。
- 「1 年 1 組」译**"1 年 1 组"**（保留日式"年 + 组"结构，不译"高一（1）班"）。
- 「特待生」译**"特待生"**（不译"公费生""特招生"）——燐羽是初星高等部 1 年 3 组特待生（[rinha profile §1](character-profiles/rinha.md#1-基本设定)）。

## 1.3 中等部前史

中等部时期的组合与事件是主线**回想 / 回溯**素材的主要来源，翻译时要能区分"当前时序"与"中等部回想"：

- 中等部有独立组合活动周期。**SyngUp!** 是初星中等部第一组合（见 §4.3）。
- 现役担当中至少三位有中等部前史：**手毬 / 美铃 / 燐羽**。
- **賀陽継**（燐羽姐姐）是中等部前偶像，已引退——她的前史构成燐羽当前性格的核心背景（见 §6.3）。

**翻译判断**：中等部时期台词往往语气与现在不同（更"純粋"）——燐羽 profile §2 已明确燐羽中等部性格开朗；翻译回想场景时不要用现在时序的偽悪口吻。

# 二、培育系统骨架

## 2.1 プロデュース（培育）

游戏核心玩法名，中文圈译**"培育"**（见 [`glossary.csv`](glossary.csv)）。核心机制：

- **一 P 一偶像**：每次培育周次只培育**一位偶像**。不同"学 P × 偶像"的关系是**并行世界线**（[[project-p-idol-parallel-worldlines]]），不同时成立。翻译具体串时先确认该串的"当前担当偶像是谁"，再选相应的关系条目。
- **周次**：培育有明确的起止（如若干周 / 关卡），达成或未达成设定目标后结束。
- **担当**：学 P 主要负责的偶像。"担当"作为系列惯例保留（见 §2.2 系列继承）。

**翻译判断**：

- `プロデュース` 译**"培育"**（不译"制作""生产"）。
- `担当` 译**"担当"**（沿用系列惯例，不译"负责的偶像""签约偶像"）。
- 「プロデューサー / P」全项目内固定译**"学 P"**（作为称呼；对话中角色以名字称呼时可用"制作人"）——见 [`style-guide.md §3.1`](style-guide.md#31-p-的称呼)。

## 2.2 三大培育分支：Sense / Logic / Anomaly

培育系统的三种类型 / 路线，代表不同的战略与语义方向：

| 日文原名 | 中文译名 | 定位 |
|---|---|---|
| センス（Sense） | **感性** | 靠状态、灵光、气质叠加取胜 |
| ロジック（Logic） | **理性** | 靠稳定管理、连锁、算计取胜 |
| アノマリー（Anomaly / Extraordinary） | **非凡** | 特殊路线，含 `強気` 等专属机制 |

**翻译判断**：

- 三个词在剧情、UI、攻略中都以「感性 / 理性 / 非凡」稳定出现（[`glossary.csv`](glossary.csv) 已锁）。
- 「ロジック」是**"理性"**（与"感性"对偶），不用"逻辑"——虽然"逻辑"字面更近但会破坏中文圈的对偶美感。
- 「アノマリー」译**"非凡"**，不用音译"阿诺马丽"，不用直译"异常"。
- 三大分支不是"技能类型"而是**"整个培育路径的性格"**——翻译剧情时若人物说"わたし、感性で行くから"要理解为"我要走感性路线"，不是"我要感情用事"。

## 2.3 三大属性：Vo / Da / Vi

偶像的三个能力属性：

- **ボーカル / Vo**：歌唱
- **ダンス / Da**：舞蹈
- **ビジュアル / Vi**：视觉表现（也见"形象"的译法，视上下文选短形式）

**翻译判断**：Vo / Da / Vi 缩写**中文语境中直接保留**（不译"歌 / 舞 / 视"）。全称按 [`glossary.csv`](glossary.csv)。

## 2.4 卡系统

- **P カード / P 卡**：培育卡（角色本卡），玩家培育时使用的角色载体。
- **S カード / S 卡**：支援卡，培育中提供辅助效果。

**翻译判断**：`P 卡 / S 卡` 的中文圈叫法优先于"培育卡 / 支援卡"全称（简洁性）——见 [`glossary.csv`](glossary.csv)。

## 2.5 状态词与培育资源

培育过程中的状态词（`好調` `絶好調` `集中` `好印象` `やる気` `熱意` `不調` `スランプ` `気まぐれ` `弱気` `強気` 等）已在 [`glossary.csv`](glossary.csv) 中全部登记，本文件不重复登记译名。

培育资源：`Pポイント / P 点数` `AP / 活动点数` `ジュエル / 宝石`（付费货币）`マニー / 金钱`（游戏内货币）。

**翻译判断**：状态词大部分**中日字形相同**（`集中` `好印象` `不調` 等）——译文中保留字形。「やる気」译**"有干劲"**或简称**"干劲"**；「スランプ」译**"萎靡"**（音译"斯兰普"不采纳）。

## 2.6 才能（才能）

学马仕作品的核心概念之一。**「偶像才能」**是一种可被识别、被培养、被评价的属性；十王星南的特技是**"看出偶像才能"**——这是驱动"星南 × 琴音"关系线的核心机制（[`relationships.md 十王星南 × 藤田琴音`](relationships.md#十王星南--藤田琴音)）。

**翻译判断**：

- `才能` 中日字形相同，译文保留字形。
- 「才能を見抜く」译**"看出才能"**（不译"发掘才华""识别天赋"，会丢掉"看穿"的精准感）。
- 「才能を棒に振る」（月花斥责四音的官方台词）译**"糟蹋才能"**，保留"棒に振る"的浪费感——见 [`characters.md 白草月花`](characters.md#白草月花しらくさ-げっか)。

# 三、亲爱度机制

## 3.1 系统名与剧情类型名的分离

学马仕的角色情感深化系统在**系统 UI** 与**剧情类型名**中使用不同的称呼：

- **系统名（好感度）**：官方 UI 与数值系统一律用「好感度」。
- **剧情类型名（亲爱度剧情）**：**亲爱度コミュ**在社区（含日文圈与中文圈）中广泛使用，尤其是二创圈——如 Research-002 §1.2 的「賀陽燐羽親愛度コミュ妄想 SS」。中文圈的 B 站翻译标题也多用"亲爱度剧情"。

**翻译判断**：

- 涉及系统数值或 UI 时用**"好感度"**（如"好感度上升""好感度 Lv3"）。
- 涉及**剧情类型 / SS 分类**时可用**"亲爱度剧情"**或**"亲爱度コミュ"**（保留原字），二者并存不冲突。
- 见 [`glossary.csv`](glossary.csv) `好感度` 与 `親愛度` 两条并存的处理方式。

## 3.2 好感度 Lv 与剧情解锁

好感度分等级（Lv1 → Lv2 → …），每达到一定等级解锁对应的角色剧情单元（`コミュ`）。

**翻译判断**：

- `Lv` 中文语境**保留字母**（如"好感度 Lv1"），不译"等级 1"或"1 级"。
- `コミュ`（communication 的日文简称）译**"剧情"**（[`glossary.csv`](glossary.csv) 已锁），不译"沟通""交流"。

# 四、组合与个人活动

## 4.1 组合与个人活动的并行

学马仕的偶像既有**组合活动**也有**个人活动**（SOLO 曲、单人卡剧情等）。两条线并行推进：

- **组合活动**：多人组合的合唱、合舞、群像剧情。
- **个人活动**：以担当偶像为中心的独立剧情线（好感度剧情多属此）。

**翻译判断**：翻译时要区分"这个偶像现在是以组合成员身份出现，还是以个人身份"——组合场景中偶像间的互动语气可能与个人剧情不同。

## 4.2 现役组合

三个稳定的组合（截至 2 周年）：

| 组合名 | 学园 | 构成 |
|---|---|---|
| **Re;IRIS** | 初星 | 花海咲季 × 月村手毬 × 藤田琴音（1 年 1 组，学 P 促成） |
| **Begrazia** | 初星 | 十王星南 × 秦谷美铃 × 花海佑芽（跨年级） |
| **極月学园四人组** | 極月 | 白草月花 × 白草四音 × 貌陽燐羽 × 藍井撫子（详见 [`relationships.md 白草四音 × 賀陽燐羽 × 藍井撫子`](relationships.md#白草四音--賀陽燐羽--藍井撫子)） |

组合与角色的具体关系见 [`relationships.md 三、Re;IRIS`](relationships.md#三reirisrira年-1-组的现役组合) 及后续区。

**翻译判断**：

- 组合名 `Re;IRIS` `Begrazia` `SyngUp!` **保留原文**，不中文化（见 [`glossary.csv`](glossary.csv) 与 [`translation-decisions.md`](translation-decisions.md)）。
- 極月学园四人组**没有官方组合名**——若原文用「極月学園の四人」「極月組」「極月四人衆」等描述，按上下文译作**"極月学园四人组"**或**"極月组"**（保留"極月"原字），不主动"命名"该组合。

## 4.3 SyngUp!（解散组合，中等部前史）

**SyngUp!** 是初星中等部第一组合，由**月村手毬 × 秦谷美铃 × 貌陽燐羽**组成，燐羽任队长。因"喧嘩別れ"（争吵）解散，三人现今分属：手毬（Re;IRIS）、美铃（Begrazia）、燐羽（極月学园）。

**翻译判断**：

- `SyngUp!` **保留原文不中文化**（见 [`translation-decisions.md`](translation-decisions.md)）。「震升!」「声扬!」等中文化尝试**一律不采用**。
- 社区自创的 `SyngUp! = Sin+ing` 七宗罪解读**不入官方设定**（见 [`community-expressions.md`](community-expressions.md) `SyngUp! = Sin+ing`），翻译官方剧情时不主动引入。
- SyngUp! 三人相互间的具体关系（青梅竹马 / 单向倾慕 / 再会等）见 [`relationships.md 二、SyngUp! 关系`](relationships.md#二syngup-关系中等部第一组合的解散与再会)。

# 五、P × 偶像的契约模型

## 5.1 培育契约的核心机制

- **一 P 一偶像 / 一周次**：每个 プロデュース 周次培育**一位偶像**（见 §2.1）。
- **契约期间**：有明确的起止时间与阶段目标（如中间试镜 / 最终试镜）。
- **契约达成 / 未达成**：影响结局与后续叙事分支。

## 5.2 并行世界线

**核心**：不同"学 P × 偶像"的关系是**并行世界线**，不同时成立。

- 一个玩家在多个 playthrough 中可以培育不同的偶像，每次都产生"该偶像的担当学 P"这一角色，但**这些"学 P"不是同一个人**（叙事上是并行世界线）。
- 组合层（如「学 P 促成 Re;IRIS」）可以**跨世界线共通**，但**组合内个别偶像与学 P 的私人关系**（如"莉波与学 P 是青梅竹马""広对学 P 说"すき"" 等）**仅在该偶像的世界线内成立**。

**翻译判断**：

- 翻译具体串时**必须先确认该串的"当前担当偶像"**（应在 `context-pack.md` 中显式写明）。
- 只启用当前担当偶像的对应关系条目；其他偶像的"学 P × ○○" 信息**不引入**当前译文。
- 详细的世界线间约束见 [`relationships.md 九、制作人（学 P）关系`](relationships.md#九制作人学-p-关系)。

## 5.3 学 P 的身份

- 学 P 在系列中是**新人 / 学生身份**——不是既成的资深制作人，而是"在学园中被培育出来的制作人"（这也是"学マス"作品名的核心创新）。
- 与学园的关系：学 P 隶属于学园的制作人培养体系。学园长十王邦夫是学园的**最终决策者**。
- 与 学生 / 偶像之间的距离：学 P 与担当偶像同年龄段（1 年生偶像 = 15 岁），叙事距离比"资深制作人 + 未成年偶像"更近。

**翻译判断**：

- 学 P 的口吻不应译成"资深大叔式的稳重"——他 / 她是同龄伙伴 + 责任担当者的复合形象。
- 「プロデューサー」被角色以敬语称呼时（如篠泽広的「プロデューサーはひどい」）不软化敬语——见 [`characters.md 篠澤広`](characters.md#篠澤広しのさわ-ひろ)。
- 学 P 与其他学园的教职员（学园长、班主任、训练员）**保持敬体**——但对同龄偶像可用平体（视关系）。

# 六、主线关键事件（时序）

以下事件为**2 周年前主线剧情反复被提及的关键节点**。只写事件名 + 一句最小必要说明 + 翻译判断锚点。**不复述剧情**，详细情节见对应角色 profile 或 relationships.md。

事件按**推定时序**排列（回想 → 现在）。

## 6.1 賀陽継 H.I.F 优勝 → 一番星（前史，中等部之前）

- **事件**：燐羽的姐姐 **賀陽継** 于 H.I.F 获优胜，获**「一番星」**称号。
- **翻译判断**：
  - `一番星` 保留日文汉字，作为**称号**处理，不译"最亮的星"。
  - 賀陽継作为"前一番星"的形象是燐羽性格转折的核心背景（见 §6.3 与 [`rinha profile §2`](character-profiles/rinha.md#2-官方形象与性格核)）。

## 6.2 中等部 SyngUp! 结成与解散（回想）

- **事件**：手毬 × 美铃 × 燐羽组成 SyngUp!；因"喧嘩別れ"解散，三人疏远（详见 §4.3 与 [`relationships.md 二、SyngUp! 关系`](relationships.md#二syngup-关系中等部第一组合的解散与再会)）。
- **翻译判断**：
  - `喧嘩別れ` 译**"争吵解散"**或**"闹翻分手"**（保留"別れ"的决裂感）。
  - 回想场景中三人语气与现在不同（更"純粹"）——燐羽 profile §2 已明确说明。

## 6.3 賀陽継 引退（前史）

- **事件**：継在获「一番星」后**引退**。原制作人推测为十王邦夫（[`relationships.md 賀陽継 × 賀陽燐羽`](relationships.md#賀陽継--賀陽燐羽)）。
- **翻译判断**：
  - `引退` 译**"引退"**（中日字形相同），不译"退役""退休"——后二者不符合偶像业界惯用词。
  - 継的引退与燐羽性格转折的关联在 [`rinha profile §2`](character-profiles/rinha.md#2-官方形象与性格核)。

# 七、系列继承与学马仕差异

## 7.1 系列继承（默认沿用）

对偶像大师主系列（765PRO / 灰姑娘 / 百万现场 / SideM / 闪耀色彩）已形成的中文惯例，**默认继承**：

- **制作人 → 学マス 中作为「学 P」**（学马仕明确不同）——见 §2.1、[`style-guide.md §3.1`](style-guide.md#31-p-的称呼)。
- **偶像**、**担当**、**亲爱度 / 好感度** 等**沿用系列共识**，不重复登记。

## 7.2 学马仕明确差异（另行锁定）

以下点是学马仕**明确与主系列不同**的设定 / 术语，翻译时必须切换：

| 概念 | 系列惯例 | 学马仕 |
|---|---|---|
| P 的称呼 | プロデューサー | **学 P** |
| 培育系统类型 | 无三分支 | **Sense / Logic / Anomaly** 三大分支 |
| P × 偶像关系 | 多偶像可并存于同一时空（如 CG、ML 群像） | **一 P 一偶像 / 并行世界线** |

其余系列继承点不重复登记，遇到冲突时立条到 [`translation-decisions.md`](translation-decisions.md)。
