# 学马仕翻译决策速查

本文件只保留**最终怎么译**。理由、来源、调研过程不放入默认翻译上下文；需要追溯时查 `docs/` 归档或 git 历史。

## 全局裁决

| 原文 / 场景 | 译法 | 禁用 / 注意 |
|---|---|---|
| `学園アイドルマスター` | 学园偶像大师 | `学马仕` 只作项目内部简称，不作为正式译名 |
| `プロデューサー` / `P`（角色称呼或叙述身份） | 学P | 对话正文中若角色固定称呼为「制作人」可按口吻使用；字幕 role 仍用 `学P` |
| `プロデュースする` / `プロデュースさせてください` 等 | 当……的制作人 / 负责……的制作 / 培育…… | 对象是人时禁用「制作你」「制作某某偶像」这类机械直译；按中文台词功能改写 |
| 担当偶像中文名 | 以 `glossary.csv` 为准 | 不在单串内另起译名 |
| `SyngUp!` | SyngUp! | 不译「震升!」「声扬!」等中文化名 |
| `Re;IRIS` | Re;IRIS | 保留分号 |
| `H.I.F` / `N.I.A` | H.I.F / N.I.A | 保留点号和英文缩写 |
| `一番星`（称号） | 一番星 | 不译「最亮的星」「第一颗星」 |

## 角色名裁决

| 原文 | 译法 | 禁用 / 注意 |
|---|---|---|
| `藤田ことね` | 藤田琴音 | 不用「藤田言音」 |
| `ことねちゃん` 等派生称呼 | 以「琴音」为基础处理，默认可用「琴音酱」 | `ちゃん` 默认不要机械译成「小〇〇」 |
| `まりちゃん` | 毬酱 | 不用「小手毬」 |
| `りんちゃん` | 燐酱 | 不用「小燐羽」 |

## 语气与常用短句裁决

| 原文 / 场景 | 优先译法 | 禁用 / 注意 |
|---|---|---|
| `え` | 欸 / 诶 | 按角色语气和惊讶强度选择 |
| `へぇ` | 哎 / 诶 | 冷淡、挑衅、感兴趣时可分别调整 |
| `まあ` | 嘛 | 多数对话场景不要默认译成「嗯」；若是认真承认或整理话题时可另行处理 |
| `ハァ` | 哈啊 / 唉 | 叹息偏重可用「哈啊」，疲惫或认命可用「唉」 |
| `ちょっと` | 等下 / 喂 | 不固定译「稍微」；按打断、抗议、喊停语境处理 |
| `もし約束を破ったら、殺すから` | 要是敢违约的话，小心我杀了你 | 燐羽威胁语气保留，允许比直译更像中文台词 |
| `痛いところついてくるじゃない` | 你还真会戳人痛处啊。 | 保留被戳中后的不爽和承认感 |
| `いいわよやってやるわ` | 行吧我就做给你看吧 | 保留让步 + 逞强，不译得太平 |
| `ハァ、わかった。` | 唉，知道啦。 | 根据上下文也可用「哈啊，知道啦」 |
| `そういう目的はあるでしょうね。` | 会有这种目的也不奇怪呢。 | 允许加「呢」保留轻微让步/揶揄 |
| `安心してください` / `安心して` | 你就放心吧 / 放心吧 | 不必总译成硬邦邦的「请放心」 |
| `骨の髄まで味わいなさい！` | 你可给我好好看清楚了！ / 可别眨眼，好好看着吧！ | 舞台挑衅语境优先译出台词功能，不必死守「骨髓」字面 |
| `貴方ってほんと私のこと好きすぎない？` | 你也真是太喜欢我了吧？ | 比「你这个人真的太喜欢我了吧？」更顺 |
| `プロデューサーの嘘つき。契約違反で殺すわ。` | 制作人你这骗子。……这可是违约，小心我杀了你。 | 不译成法律腔的「你违反契约，我要杀了你」 |

## 叙述句与抽象比喻裁决

| 原文 / 场景 | 优先译法 | 禁用 / 注意 |
|---|---|---|
| `彼女の喉から、伴奏も何もない、完全な歌声が放たれる。` | 从她喉中迸发出的，是没有任何伴奏、纯粹无瑕的歌声。 | 不译「从她喉咙里释放出没有伴奏的完整歌声」；`完全な歌声` 在舞台/歌唱描写中优先按“自足、无瑕、已然成立的歌声”处理 |
| `歪みのない美しさ` | 无瑕之美 / 毫无破绽的美 / 美得毫无破绽 | 不译「没有歪斜的美」「没有一丝歪斜之美」 |
| `目詰まりを起こす`（抽象滤镜、才能、思维、歌声等） | 堵住 / 卡住 / 失灵 | 不译「发生堵塞」；不是物理滤网时不要把中文写得过于器械说明 |
| `型崩れを起こす`（歌声、表演、人格面具、模仿等） | 走样 / 失了形 / 崩坏 | 不默认译「散架」；若主语是歌声或表演，要选中文里能搭配抽象对象的动词 |
| `みるみるうちに` | 转瞬之间 / 肉眼可见地 / 眼看着 | `眼看着` 后面必须接自然中文动作，如“变形、崩塌、走样”；不要接“就要散架”这类会让抽象主语突兀的说法 |

## 使用边界

- 本文件只覆盖**反复出现且容易译乱**的裁决。
- 角色口吻看 `characters.md` / `character-profiles/*.md`。
- 人物关系看本串相关的 `relationships.md` 条目。
- 剧情背景只在原文明确提到事件、系统或专名时查 `common-background.md`。
- 社区绰号、二创流派、黑话只在原文实际出现时查 `community-expressions.md`，不要默认加载。
