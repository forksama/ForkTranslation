# 学马仕翻译决策登记

- 建档：2026-07-13 · 首版
- 上游规格：[Research-005 §9](../../../docs/research-005-gakumas-stable-knowledge-first-version-spec.md#9-translation-decisionsmd-首版规格)

本文档登记学马仕翻译项目中**反复出现、有分歧、或容易被下次翻译推翻**的翻译决策。目的是让"以前定过的东西"不被重新讨论。

- **不重复登记**已在 [`glossary.csv`](glossary.csv)（专名译法）与 [`style-guide.md`](style-guide.md)（风格规则）中锁定的内容——本文件只登记"元决策 / 原则性决策"，具体译名与风格规则**引用**它们，不复制。
- **首版是空壳**：只预填几条已确认的锚定条目 + 登记规则。实际翻译中出现的**新决策**按 §3 的登记模板添加。

## 1. 使用规则

### 1.1 入库门槛

**不预先设定入库门槛**。是否登记由每条决策的性质单独判断：

| 情况 | 是否入库 |
|---|---|
| 一次性局部选择（如某串某句译文的微调） | ❌ 不入库 |
| 无争议的惯例（如系列继承的默认译法） | ❌ 不入库 |
| 反复出现的表达 / 多篇内容都会遇到 | ✅ 入库 |
| 有多种译法且社区存在分歧 | ✅ 入库 |
| 容易被下次翻译推翻的临时判断 | ✅ 入库 |
| 违背系列惯例的学马仕特有决策 | ✅ 入库 |

### 1.2 scope 字段

每条决策必须标明作用域：

| scope | 含义 | 举例 |
|---|---|---|
| `global` | **全领域适用**，任何串都应遵守 | `SyngUp!` 保留原文 |
| `character` | **只对指定角色生效**（角色名 / 角色专用称呼 / 角色专用口癖译法） | `藤田ことね → 藤田琴音` |
| `domain-thread` | **只对指定串生效**（一次性局部决策，避免被误当全领域） | 某具体 SS 串内的临时称呼固定 |

`domain-thread` 决策通常应放在该串的 `context-pack.md` 里；只有当**局部决策可能被误传到其他串**时才在此文件登记以隔离。

### 1.3 与其他文件的分工

| 内容 | 归属 |
|---|---|
| 具体专名的中文译名 | [`glossary.csv`](glossary.csv) |
| 风格规则（引号、语气词、粗口强度等） | [`style-guide.md`](style-guide.md) |
| 角色说话风格与固定口癖 | [`characters.md`](characters.md) / `character-profiles/*.md` |
| 社区表达、黑话、二创流派名 | [`community-expressions.md`](community-expressions.md) |
| 官方设定与背景骨架 | [`common-background.md`](common-background.md) |
| **原则性决策 / 元决策 / 跨文件锚定** | **本文件** |

**冲突处理**：当本文件的决策与其他文件的登记有冲突时，以本文件为准（本文件是"最终裁决"层），并同步修订另一文件对齐。

## 2. 首版预锁定决策

以下决策在项目启动阶段（2026-07-13）已确认。新决策的登记见 §3 模板。

### 2.1 `SyngUp!` 保留原文

- **scope**：`global`
- **决策**：`SyngUp!` **保留原文不中文化**。不采用「震升!」「声扬!」等中文化尝试。
- **理由**：
  - 该词是中等部第一组合的**作品命名**，是学马仕世界内的专名，中文化会丢失原字的品牌识别度。
  - 交叉验证多个中文来源（[`sources.md`](sources.md)），未见任何权威 UP 主 / 汉化组尝试中文化。
  - 社区自建的解读（`SyngUp! = Sin+ing` 七宗罪）在 [`community-expressions.md`](community-expressions.md) 中登记为**社区解读**，非官方，不影响本决策。
- **实施**：[`glossary.csv`](glossary.csv) 已登记（`SyngUp!,SyngUp!,group,confirmed`）。
- **确定日期**：2026-07-13

### 2.2 `藤田ことね → 藤田琴音`

- **scope**：`character`（藤田ことね 一角色）
- **决策**：`藤田ことね`（平假名原名）在**全项目内统一译作「藤田琴音」**。**不使用「藤田言音」等其他译法**。
- **理由**：
  - 「琴音」是 GameKee 与 gakumas.cn 项目组 + 独立 UP 主（学园偶像大师同好会、Anna__P 等，见 [`sources.md`](sources.md)）采用的主流译名。
  - 「言音」为少数来源的另译，与主流不一致，且用户明确指示不采用。
  - 平假名「ことね」按 `こと=琴`、`ね=音` 分解与"琴音"呼应，声调与字形均自然。
- **实施**：[`glossary.csv`](glossary.csv) 已登记（`藤田ことね,藤田琴音,character,confirmed`）。
- **确定日期**：2026-07-13
- **例外**：无。所有语境中「ことね」相关的独立指代（含缩写「ことねちゃん」的中译）都以"琴音"为基础。串内自造别名（如社区绰号"皮卡丘"）见 [`community-expressions.md`](community-expressions.md)。

### 2.3 担当名锚定原则

- **scope**：`global`（元决策）
- **决策**：**所有担当偶像的中文名一旦在 [`glossary.csv`](glossary.csv) 中确定，即为该角色在全项目内的统一译名，不允许某个串内单独变体**。
- **理由**：
  - 学马仕无官方中文版，所有译名来自社区。若允许各串自造译名会导致同一角色在不同翻译产品中呈现不同名字，破坏读者识别。
  - `glossary.csv` 是全项目**唯一**的角色译名裁决源；本文件是**执行原则**的登记，不是重复译名。
- **实施**：
  - 具体角色译名以 [`glossary.csv`](glossary.csv) 的 `character` 类条目为准。
  - 单串内如果对角色名需要临时变体（如剧情需要保留日文原名、角色自称的假名转写等），必须在**该串的 `context-pack.md` 里显式说明**，不允许"默认"使用变体。
- **确定日期**：2026-07-13
- **相关**：见 [Research-005 §7.3 冲突登记](../../../docs/research-005-gakumas-stable-knowledge-first-version-spec.md#73-冲突登记) 与 [`sources.md §4 权威等级与冲突裁决`](sources.md#4-权威等级与冲突裁决)。

### 2.4 `プロデューサー / P → 学 P`（引用锁定）

- **scope**：`global`
- **决策**：`プロデューサー / P` 作为**角色称呼**固定译作 **`学 P`**。
- **实施**：已在 [`style-guide.md §3.1`](style-guide.md#31-p-的称呼) 锁定——本条目**只作引用**，不重复规则细节。
- **理由摘要**：主系列译作"制作人"；学马仕作为"学园中被培育的制作人"是作品核心创新，需要与系列区分。
- **例外**：对话正文中角色以名字或亲近方式称呼 P 时可用"制作人"或角色的固定称呼——详见 [`style-guide.md §3.1`](style-guide.md#31-p-的称呼)。
- **确定日期**：2026-07-13（沿用项目启动约定）

### 2.5 `学園アイドルマスター` 项目内简称 —— **首版不做决定**

- **scope**：（待定）
- **决策**：**首版不锁定**。「学马仕」作为项目内工作简称已在文档系统中使用，但**是否作为翻译产品中的稳定简称**，等实际翻译中出现分歧再补。
- **理由**：
  - 项目内文档使用"学马仕"是**工作性表达**，不代表翻译产品用它。
  - `学園アイドルマスター` 在正式翻译产品中一般应译作**"学园偶像大师"**（[`glossary.csv`](glossary.csv) 已锁 `content` 类）。
  - 日文简称`学マス`是否有对应中文简称，社区目前有并用现象（"学马仕"/"学园偶像大师"/"学マス"），未见一统。
- **待定条件**：**实际翻译过程中若同一串内出现"学园偶像大师"与简称并用的必要**，届时立条决定。

## 3. 新决策登记模板

新增决策时，按以下模板在 §4 追加。**每条决策必须包含所有字段**。

```markdown
### 2.X <一句话决策概括>

- **scope**：`global` / `character` / `domain-thread`
- **决策**：<具体决策内容，明确到"选什么、不选什么">
- **理由**：<为什么这么定,含调研来源、社区共识、上下游约束>
- **实施**：<在哪里落地,如 glossary.csv 的具体条目 / style-guide 的具体章节 / 本文件唯一登记>
- **例外**：<有无例外情形;无则写"无">
- **确定日期**：<YYYY-MM-DD>
```

登记原则：

- **决策**字段用**祈使/断言语气**，不用"建议""可以""通常"等模糊词。
- **理由**字段必须包含**至少一个可核查的证据**（引用 sources.md 中的来源、多源交叉的具体产出、或用户明确指示）。
- **实施**字段必须**指向具体条目**（`glossary.csv` 的某行 / `style-guide.md` 的某节 / 本文件唯一登记）。
- **例外**字段若非"无"，必须**穷举**——不能写"视上下文"这种滑动条款。

## 4. 后续决策（待增补）

首版仅登记 §2.1–2.5 五条。以下情境常在实际翻译中触发新决策，遇到时按 §3 模板增补：

- 具体角色对特定人物的**固定称呼**（如「まりちゃん」是保留日文还是本土化——见 [`community-expressions.md まりちゃん`](community-expressions.md#まりちゃん--月村手毬)）。
- **组合内成员间的相互称呼**（如 Re;IRIS 三人如何互称、SyngUp! 时期与现在的称呼差异）。
- **`〜ちゃん` 的例外保留**（style-guide §3.2 允许在特定语境保留"〇〇酱"，具体判定阈值需实际串内确认）。
- **日式书名号与作品名的处理边界**（[`style-guide.md §6.4`](style-guide.md#64-书名号--作品名号) 已锁定"按原文保留"，但某些跨文本引用场景可能需要单独裁决）。
- **`一番星` 的处理**（[`community-expressions.md「H.I.F 一番星」`](community-expressions.md#hif-一番星) 已建议保留日文汉字作为称号，但作为**普通描述**时是否可意译，需实际翻译确认）。
- **社区绰号的正文进入门槛**（如"皮卡丘""毛球"等在正文译文中出现的判定阈值，见 [`community-expressions.md 一、角色社区绰号`](community-expressions.md#一角色社区绰号)）。
- **系列继承点的冲突单独立条**（[`common-background.md §7.2`](common-background.md#72-学马仕明确差异另行锁定) 已列三条明确差异，其他系列继承点若在学马仕出现分歧，登记于此）。
